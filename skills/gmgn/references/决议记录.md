# 模板 · 决议记录

## 适用时机

出现需留痕、日后会被引用的裁决时——评审意见处置、方案冻结、推翻既有结论、拍板取舍。任何"这事最终怎么定的、为什么"的决定，往账本追加一节，不散落在各处对话里。

## 模板正文

```
---
目标: <本账本记什么范围的裁决>——每条裁决唯一编号 §N 供全仓溯源
上游: <裁决依据的上游，如评审意见、设计清单、需求池>
下游: <裁决落地的产物，如规格文档各版本修订>
状态: 滚动只增——<已关账区间> / <滚动区间>
类型: 决议记录
---

# <范围>裁决账本

> 裁决口径：**采纳**（进修订）/ **采纳-改**（方向对，按批注收敛）/ **驳回**（附理由留痕）。
> 只追加不改写：§N 一旦分配不复用、不回收；已落节不改写，需修正另起新节引用旧节。

## §<N> <一句话主题>（<裁决者>，<日期>；<触发场景>）

**背景**：<一句话说清为何需要这次裁决——争议点或触发来源>。

**裁决**（原文，逐字域，后续引用只可"摘要一行 + 链接"、禁改写）：

> <裁决正文逐字。处置外部意见的标口径：采纳 / 采纳-改<按谁的批注> / 驳回<理由>>

**附带条件**：
1. <生效的前提或约束条件>
2. <…>

**传导清单**（本裁决导致哪些文档要改，改完勾选；未勾完 = 未真正生效）：
- [ ] <文档 A> §<x>：<改什么>
- [ ] <文档 B>：<改什么>

**生效状态**：<已生效 / 待某前置 / 待某会签>。

## §<N+1> …（下一条裁决，同结构往后追加）
```

## 填写说明

- **编号连续、只增不改**：§N 分配后不复用、不回收；已落节的字不改，要修正就另起新节写"§X 订正"并与旧节互链。这是账本可溯源的根基。
- **裁决原文是逐字域**：blockquote 内是权威文本；别处引用只能"摘要一行 + 链接"，禁转述改写——转述必然漂移，日后无法判断以哪个版本为准。
- **传导清单是关账钩子**：一条裁决常要改动上下游多份文档；勾选项没清完，这条裁决就还没真正落地。
- **口径三选一**：处置外部意见时标 采纳 / 采纳-改 / 驳回，驳回必附理由（否则日后无法判断当初为何不采纳）。批量意见处置可换「意见 / 要点 / 裁定 / 去向」四列表，一行一条意见，裁决仍逐字、传导仍挂。
- **反模式 · 节号被回退复用不注明**：某节内容被推翻回退后，节号被后来的裁决复用，却不加说明 → 所有指向该节号的旧引用全部落到了错误内容上。回退复用必在节首显式注明（旧用途、回退经过、现行指针）。
- **反模式 · 理由与结论分离**：只把结论记进账本、理由写在别处（或不写）→ 日后无法判断裁决前提是否仍成立。理由随裁决同节落。
