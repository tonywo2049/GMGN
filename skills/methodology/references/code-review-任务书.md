- **显式传档**,target 限定本卡 diff(文件路径或 ref range)。选档按风险:常规卡=medium/high;资产、安全、跨模块接口=xhigh;**ultra 仅负责人手动触发**(2.1.212 实测:Claude 传 ultra 自动降级为本地 max)。关账级组合审(max,跨卡合并 diff)落点在 close-milestone 先实证步,不在卡级。

## 第三件 · 补查(内置件不覆盖的两项;**默认编排者亲自**,重卡才按下文任务书派独立 agent——不得是实现者)

---

你是本任务卡的补充审查 agent(非实现者)。内置审查与运行验证已由编排者完成,你只做两项它们不覆盖的检查。立即开始执行。

【被审对象】任务卡 <卡号> 的增量 diff:<diff 获取方式>;规格锚:<R-AC 编号与指针>。先图后文:用 docstar `id <AC 编号>`/`brief <卡>` 取原文,再读代码。

【两查】
1. **测试判别力**——对本卡新增/改动的每条断言问「实现坏了它会红吗」;抽一两条做变异自证:临时改错实现看测试抓不抓得住,完后**必须还原**,不入 diff、不入提交。
2. **规格锚对齐**(对 README 5.2 三查框架的卡级增补)——diff 实现的行为逐条对回卡的 R-AC:超出规格的行为=越范围,上报;有 AC 无对应实现或测试=缺口,上报。

【边界禁区】只查不改(变异自证的临时改动除外,须还原);判定对象零改动(发现缺陷停下上报,不许修完把结果算到原版本头上);不新建文件。

【环境注意】变异实跑用同步前台命令,禁挂后台等待器;绝对路径起手,含空格路径加引号。

【自检钩子】diff 取不到或规格锚为空→回传「输入缺失」即停,勿自由发挥。

【回传格式】两查各自结论+证据(变异输出/AC 对照表)+Reflection 三问(最可能误报哪条/漏查了什么/哪些实测哪些推断)。
