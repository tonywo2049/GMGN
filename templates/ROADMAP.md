# 模板 · ROADMAP（节奏权威）

## 适用时机

项目需要一份「把系统全集切成阶段、逐段给出目标」的总节奏图时——全项目**唯一一份**，在时序上先于任何 Milestone 的需求池。只承载什么、不承载什么见下方骨架定位。

## 模板正文

拷贝围栏内整段骨架，按 `<...>` 逐处替换；固定小标题保留。头部元信息块（YAML 五字段）见《文档元信息块》模板，类型填「ROADMAP」。

````
# <项目名> ROADMAP（唯一节奏权威）

**定位**：本文是 <项目名> 的**唯一节奏权威**。**只承载「阶段划分与阶段目标」**——把系统全集切成里程碑（Milestone），逐个给出一句话阶段目标与状态。**具体交付物、退出判据、需求一律不在本文**：它们在各 Milestone 立项时于该 Milestone 内约定（节奏权威在时序上先于任何具体需求——写它的时候验收标准还不存在）。

**需求按 Milestone 分池，无全集需求池**：每个 Milestone 立项时自建**只针对本 Milestone 的需求池**；不存在覆盖所有 Milestone 的大需求池。全集层面的约束在**目标层**（架构目标与顶级不变量，回答「必须做到什么」）——它才是「覆盖全面」的那一层。

**不设日历时间盒**：一切里程碑以完成判据关账，无日历死线；防漂移靠范围纪律与门禁（见 §3 状态机与立项规则），不靠排期。

**粒度约定**：Milestone = 大节段划分（版本级）；Milestone 内按**切片**细分，切片是立项与状态机的最小单元（§3）。

## 2. TODO 待分配（有想法、尚未规划进 Milestone）

以下事项**已有想法但还没具体规划**，等待被分配进某个 Milestone。**分配由负责人裁决**；分配进某 Milestone 后，在**该 Milestone 内成文需求**（建其自有需求池）再执行，不跳过登记直接空降一个执行套件。

1. **<事项一句话>**——<内容摘要>；来源：<从哪来，如某上层目标章节 / 某次评审>；既有裁决约束：<已拍板、不可推翻的前提，如某方案已决不变>。
2. **<...>**——<...>。

## 3. Milestone/切片状态机与立项规则

**状态机**：`未立项 → 立项 → 进行中 → 关账（已完成）`。

**立项规则**（切片开工前置）：① 在其执行套件登记「立项」；② 建目标-需求-设计-任务套件（先立 Goal 一句话定位 → 需求成文 → 负责人批 → 任务立卡）。**未立项切片的任何派工 = 越范围，会话内授权不替代立项。**

**变更规则（两轨）**：

- **存量轨**（早于本体系已成池的需求基线）：变更走该基线文档 + 分配账本的**受控演进**（接缝 / 会签 / 收账基准，不新造机制）。
- **增量轨**（新想法 / 新业务线）：走 **§2 TODO 登记 → 负责人分配进某 Milestone → 该 Milestone 自建需求池成文 → 执行**；不经存量分配账。

**唯一叙述点**：切片的实时状态、波次细账、验收标准的测试覆盖、契约冻结层级均在各执行套件 / 追踪矩阵 / 冻结索引查，本文不复述——指针到位即可。
````

## 填写说明

- **定位声明是本文的护栏**：四句（唯一节奏权威 / 只承载阶段划分与阶段目标 / 不设日历时间盒 / 粒度约定）钉死本文边界，挡住「把判据、需求、台账往里塞」的冲动。
- **总图无判据列**：状态只用状态机四词，退出判据不在此——写 ROADMAP 时它们还不存在。反模式：在总图里加「退出判据」「验收标准」列，或把某 Milestone 的切片台账搬进来（切片实时状态归各执行套件，本文只留指针）。
- **TODO 三要素缺一不可**：一句话 + 来源 + 既有裁决约束。缺「来源」则日后无法判断这想法从哪来；缺「既有裁决约束」则分配时可能推翻早已拍板的前提。
- **两轨变更规则别混**：存量需求（早于本体系的基线）走分配账受控演进；新想法一律走 TODO → 负责人分配 → Milestone 内成文。
- **反模式 · ROADMAP 承载验收标准分配**：把「哪条验收标准归哪个 Milestone」写进本文——那是分配账本（存量映射账）的职责，不是节奏权威的。ROADMAP 只到 Milestone 目标级，不下探到具体验收标准。
