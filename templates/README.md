# 模板库

本目录收方法论配套的可拷贝模板。新项目启动动作之一，就是把这些模板拷进去、按《从零启动清单》（见上级 `README.md` 第 8 章）过一遍。

**当前状态**：本文件只列模板清单与每项的一句话内核，暂不含模板正文。模板本体待某个使用项目完成结构改造后，从真实文档形态提炼，另批确认。在那之前，请以上级 `README.md` 的对应章节为准。

## 十三项模板清单（「核对单」一项捆绑三张，物化后共十五个文件）

- **文档元信息块**——每份文档头部的五字段（目标·上游·下游·状态·类型），让任意一篇能沿"上游"逐层回溯到需求根。
- **需求文档**——唯一需求源，每条需求配可判定、唯一编号的验收标准，数值参数化。
- **设计文档**——把需求映射到实现对象，回答"用什么结构满足需求"。
- **任务文档**——执行权威，任务卡挂规格锚、失败先行测试、完成判据，配滚动台账。
- **ROADMAP**——唯一节奏权威，只写阶段划分与各阶段目标（阶段级语言，不承载验收标准分配；它在时序上先于需求全集）。
- **分配账本**——"验收标准 → Milestone"矩阵，需求全集出现后建立、与需求池同置；无归属的显式进 backlog。
- **决议记录**——追加式账本，记裁决、附带条件、传导清单，只追加不改写。
- **交接 handoff**——接手横幅，一句话给出状态、基线、剩余、指针。
- **跨轨登记册**——登记并行各线的门禁与决策点，只登记不复述。
- **派发与回传**——派发任务书（必读锚、边界禁区、模型与档位选择）与回传格式（含 Reflection 段）。
- **核对单三张**——上机前、合并前、关账前，各一张时机绑定的开放式问题清单。
- **信任面登记表**——新增或改动"消费外部输入并改变状态"的入口时，逐行核的表。
- **最薄弱假设呈报**——关账或裁决呈负责人时附的三行：最弱假设｜若塌会怎样｜证据强度与最便宜的证伪动作。
