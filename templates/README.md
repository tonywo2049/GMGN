# 模板库

本目录收方法论配套的可拷贝模板。新项目启动动作之一，就是把这些模板拷进去、按《从零启动清单》（见上级 `README.md` 第 8 章）过一遍。

**当前状态**：十六个模板文件已全部成文（十四项，其中「核对单」捆绑三张），状态均为「草稿-待负责人确认」。使用时**以各模板文件为准**；本文只留清单与每项一句话内核 + 指针。

## 十四项模板清单（「核对单」一项捆绑三张，物化后共十六个文件）

- **[文档元信息块](文档元信息块.md)**——每份文档头部的五字段（目标·上游·下游·状态·类型），让任意一篇能沿"上游"逐层回溯到需求根。
- **[目标文档（Goal）](目标文档.md)**——阶段/Milestone 工作单元的入口文档，标准文件名 `Goal.md`，兼并该单元的 README 职能：定位、阶段目标、边界句、切片划分、非目标、链路文档映射、验收判据、缺口清单。未立项的事项记为缺口，立项时才建对应的需求/设计/任务文件。区别于覆盖全项目、几乎不变的顶层目标文档（见本文末「不提供骨架」条）。
- **[需求文档](需求文档.md)**——每个 Milestone 一份需求池，立项时成文；每条需求配可判定、唯一编号的验收标准，数值参数化。覆盖全面的需求集不单独成池，由目标层的大目标承担。标准文件名 `Requirement.md`，是所在单元唯一需求权威。
- **[设计文档](设计文档.md)**——把需求映射到实现对象，回答"用什么结构满足需求"。
- **[任务文档](任务文档.md)**——执行权威，任务卡挂规格锚、失败先行测试、完成判据，配滚动台账。
- **[ROADMAP](ROADMAP.md)**——唯一节奏权威，写阶段划分、各阶段目标与 Milestone 级**定性验收标准**（完成景象的语言描述，可判不量化；负责人裁决 2026-07-18），另挂 TODO 待分配列表（记已有想法但尚未规划进任何 Milestone 的事项，等待被分配）；不承载需求级验收标准（R-AC）与量化指标（它们在 Milestone 立项后于 Requirement 内成文），它在时序上先于各 Milestone 的需求池。
- **[分配账本（可选，存量映射账）](分配账本.md)**——仅当项目在采用本体系前已有跨 Milestone 的需求存量时才用：一张"存量需求 → Milestone"矩阵，把存量逐条划入新 Milestone，一次性处置、非长期机制。无历史存量的新项目不需要它；新想法走 ROADMAP 的 TODO 待分配列表。
- **[决议记录](决议记录.md)**——追加式账本，记裁决、附带条件、传导清单，只追加不改写。
- **[交接 handoff](交接handoff.md)**——接手横幅，一句话给出状态、基线、剩余、指针。
- **[跨轨登记册](跨轨登记册.md)**——登记并行各线的门禁与决策点，只登记不复述。
- **[派发与回传](派发与回传.md)**——派发任务书（必读锚、边界禁区、模型与档位选择）与回传格式（含 Reflection 段）。
- **核对单三张**——[上机前](核对单-上机前.md)、[合并前](核对单-合并前.md)、[关账前](核对单-关账前.md)，各一张时机绑定的开放式问题清单。
- **[信任面登记表](信任面登记表.md)**——新增或改动"消费外部输入并改变状态"的入口时，逐行核的表。
- **[最薄弱假设呈报](最薄弱假设呈报.md)**——关账或裁决呈负责人时附的三行：最弱假设｜若塌会怎样｜证据强度与最便宜的证伪动作。

**不提供骨架的一类**：接口契约（高度领域化，通用骨架无承载价值，其纪律见 README 3.2）。项目级顶层目标文档（覆盖全项目、几乎不变的那一份，见 README 2.A「目标」条）已具名**白皮书（WhitePaper）**，按七项成文要求撰写（见 skills/brainstorm/SKILL.md；负责人裁决 2026-07-18，取代旧「三问自搭、不强定骨架」口径）——它与本页「目标文档（Goal）」模板不是同一层级：后者是阶段/Milestone 工作单元的入口文档，有固定骨架。「变更管理闸」「配置基线」是规则不是文档，见 README 2.C 与 3.4。
