# 模板 · 交接 handoff

## 适用时机

每次关账后立刻写（对应第 3.5 章"关账后立即写接手态"）。下一个会话从横幅进入，而不是从记忆进入。长任务分波推进时，每波收官刷新横幅。

## 模板正文

```
---
目标: <线/模块>状态交接——<最新态一句话>
上游: <任务文档、决议记录、冻结索引 等权威源>
下游: <后续会话续跑什么>
状态: 交接（<最新态一句话>）
类型: 交接 handoff
---

# <线/模块>交接（<日期> · <一句话里程碑>）

> 承接 <上一份 handoff 链接>（<上一态基线 + 一句话>）。

## 1. 一句话状态（横幅——只留最新一屏，最新先读本段）

> **<状态标记> <当前波次/阶段>收官态（<日期>）**
>
> **基线 = <分支> `<commit>`（<已 push / 已合并入主干 / 未 push>）**。
> - **当前状态**：<系统现在处于什么态，一句话>。
> - **剩余工作**：<下一步做什么、阻塞在哪>。
> - **禁区 / ⚠️**：<接手不能碰的、易踩的坑、待确认未授权项>。
> - **接手入口（按序）**：<本横幅 → 决议记录 §x → 任务文档 §y → 台账/记忆>。

## 2. <滚动正文——按波次/阶段追加的详情段>

<合并态台账、门/阻塞详情、环境、教训……横幅里只给指针，详情在此逐段展开、只往后追加。上一波横幅里仍有用的耐久事实下沉到这里，横幅本身不堆历史。>
```

## 填写说明

- **横幅四件硬性**：状态一句话 / 基线（分支 + commit hash + push 或合并态）/ 剩余 + 阻塞 / 接手入口指针链。缺一件，接手人就得翻历史自己拼，"60 秒开工"的目标就破了。
- **基线要能直接 checkout**：写分支名 + commit hash + 是否已 push/合并，别写"最新代码"这类会过期的话。
- **每次关账刷新横幅**：新态覆盖到横幅顶部；旧横幅里仍有用的耐久事实（合并记录、教训、门态）下沉到正文波次段。
- **反模式 · 横幅长成流水账**：把每一波历史都堆在横幅段里、越堆越长，接手人 60 秒读不完 = 横幅失效。横幅只留最新态一屏，历史进正文。
- **反模式 · 完成态不挂验证路径**：横幅写"已完成/已关账"却不同句写明谁在什么真实产物上验过 → 接手人不知道该信到什么程度。完成态紧挂实体验证（谁、在哪个产物、跑了什么），呼应第 1 章表征-实体脱节。
