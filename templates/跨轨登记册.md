# 模板 · 跨轨登记册

## 适用时机

多条线并行推进、彼此间有门禁 / 依赖 / 待决策点时用。它是状态链最粗的一层（任务台账 → handoff → 登记册 → 节奏权威只留指针），往上只留指针、不复述细节。单线任务不需要。

## 填写说明

- **只登记不复述**：每条只留"门 / 决策点是什么 + 指针"，细节在各线文档展开。判据可机检——本文件出现第二处完整叙述就是漂移面，删掉换指针。
- **门禁表五列**，"责任归属"最易漏：写清"谁做才解锁"，跨线门常因无人认领而长期悬空。
- **逐字引用域**：门的通过判据若来自某条裁决，用 blockquote 逐字抄原文 + 指针到决议记录，禁转述——判据一字之差可能改变门的通过条件。
- **已决项不留长文**：决策一旦拍板，正文只留一行指针到决议记录，不保留一大段划掉的历史（历史看 git）。
- **反模式 · 复述各线任务细节**：登记册里展开各线进展 → 与各线文档双份存在、必然不同步（典型漂移：登记的预设与实际相反、已冻结的仍标"延后"）。发现第二处完整叙述即删。
- **反模式 · 判据转述而非逐字**：门禁判据写"大意如此" → 在传递中变形，门的通过条件失真。判据逐字引用 + 指针。
