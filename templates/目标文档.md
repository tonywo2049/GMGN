# 模板 · 目标文档（Milestone / 阶段入口，Goal）

> 标准文件名 `Goal.md`；兼并该单元的入口/README 职能——一个目录一个入口文档。不同于覆盖全项目、几乎不变的顶层目标文档（后者形态因项目而异，不强定骨架，见 README 2.A「目标」条与本目录 README「不提供骨架」条）。

## 适用时机

某个 Milestone / 阶段级工作单元**立项时**，作为该单元目录的入口文档首先建立——先于需求（Requirement）、设计（Design）、任务（Task）文件。它同时是该单元唯一的 README，不必另建一份。

## 模板正文

拷贝围栏内整段骨架，按 `<...>` 逐处替换；固定小标题保留。头部元信息块（YAML 五字段）见《文档元信息块》模板，类型填「目标文档」。

```
# <单元名> 目标（Goal）

## 定位

<一句话：这个目录/单元是什么，解决什么>。

## 阶段目标

<本单元要在这一阶段达成什么；服务于上游哪个更大目标——指向 [目标层大目标](<指针>) 与 [ROADMAP 对应行](<指针>)>。

## 边界句

<本单元的权限边界一句话：向上不改什么（不重开上游已冻结的选型/契约），向下不越什么（不替相邻单元下定义）。若本单元产出技术选型/架构结论，标注"结论仅供下游消费，下游不得就地另选"；若本单元消费别处的选型/架构结论，标注"发现结论不支撑需求时走显式回退升级，不就地另选">。

## 切片划分

| 切片 | 一句话范围 | 状态 |
|---|---|---|
| <切片名> | <范围> | 未立项 / 立项 / 进行中 / 关账 |

## 非目标

- <显式排除的范围>——<归哪个单元 / 哪个阶段>。

## 链路文档映射

本单元 G-R-D-T 链路文档的指针，未立项的环节留空、不预先建空文件（防空抽象）：

| 文档 | 标准文件名 | 状态 | 指针 |

| 需求 | `Requirement.md` | <未立项 / 已建> | <链接或 —> |
```

## 填写说明

- **Goal 是入口不是详情**：定位/阶段目标/边界句要让人几十秒看懂"这是什么、要做到什么、边界在哪"；细节留给需求/设计/任务，Goal 里只放指针。反模式：把需求条文或设计细节誊抄进 Goal——重复内容是漂移面。
- **切片是立项粒度**：切片划分表是本单元向下立项的清单，状态列随执行推进刷新；未立项的切片不建需求/设计/任务文件。
- **链路文档映射只放指针**：每行指向对应文件或标"未立项"，不复述其内容。
- **验收判据是单元级出口判据**：回答"这一整个阶段/单元何时算完成"，不是某条需求的验收标准——那是 Requirement 层的事，不在此重复定义。
- **缺口清单先于新建文件**：想到但还没立项的事项先落一行缺口，不要因为"想到了"就顺手建需求/设计文件占位——空文件本身就是过度设计。立项时，缺口清单里对应行删除，转成切片划分表里的一行。
- **边界句接住阶段边界纪律**：本单元若是选型/架构性质，说明下游只消费不重选；若本单元消费别处的选型/架构结论，说明发现不支撑时走显式回退升级、不就地另选（见 README 2.C「阶段边界按工作性质划分」）。
